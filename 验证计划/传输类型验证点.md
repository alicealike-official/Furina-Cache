## 传输类型验证点优先级表

### 表1：HTRANS 传输类型验证点

| 优先级 | 传输类型 | 验证点 | 说明 |
| :---: | :---: | :--- | :--- |
| **P0** | `NONSEQ` | 新突发第一笔必须使用 NONSEQ | 基本协议合规性 |
| **P0** | `NONSEQ` | 地址与前一笔传输无关 | 基本协议合规性 |
| **P0** | `SEQ` | 突发后续笔必须使用 SEQ | 基本协议合规性 |
| **P0** | `SEQ` | 地址计算正确（递增或回环） | 影响数据完整性 |
| **P0** | `IDLE` | 无传输时主机必须发 IDLE | 基本协议合规性 |
| **P0** | `IDLE` | 从机必须零等待 OKAY 响应 | 基本协议合规性 |
| **P0** | `IDLE` | 从机必须完全忽略该传输 | 基本协议合规性 |
| **P0** | `BUSY` | 从机必须零等待 OKAY 响应 | 基本协议合规性 |
| **P0** | `BUSY` | 从机必须完全忽略该传输 | 基本协议合规性 |
| **P1** | `BUSY` | 只能在突发中间使用（非首/末拍） | 常见错误点 |
| **P1** | `BUSY` | BUSY 周期地址必须指向下一笔有效传输 | 影响突发正确性 |
| **P1** | `BUSY` | 固定长度突发不能以 BUSY 结束 | 协议违例检测 |
| **P1** | `IDLE` | 突发完成后应插入 IDLE | 总线空闲规范 |
| **P2** | `BUSY` | 未定义长度突发（INCR）可以 BUSY 结束 | 边界场景 |
| **P2** | `IDLE`/`BUSY` | 与 ERROR 响应隔离 | 错误处理隔离 |

### 表2：HTRANS 传输类型汇总

| 传输类型 | HTRANS[1:0] | 是否传数据 | 典型场景 | 优先级 |
| :---: | :---: | :---: | :--- | :---: |
| `IDLE` | 00 | ❌ | 主机空闲，无数据要传 | P0 |
| `BUSY` | 01 | ❌ | 突发中间，主机暂时无法提供数据 | P1 |
| `NONSEQ` | 10 | ✅ | 突发第一笔或单次传输 | P0 |
| `SEQ` | 11 | ✅ | 突发后续笔 | P0 |

| | 控制信号 | HWRITE/HSIZE/HBURST 等正确设置 | P0 |
| | 从机响应 | 可插入等待，返回 OKAY/ERROR | P0 |

| | 地址计算（回环） | WRAP 类型在边界处正确回绕 | P0 |
| | 控制信号稳定性 | 突发内 HWRITE/HSIZE/HBURST 不变 | P0 |
| | 从机响应 | 可插入等待，返回 OKAY/ERROR | P0 |

### 表4：验证模式与传输类型对应关系

| 验证模式 | 涉及的主要传输类型 | 核心验证点 |
| :---: | :--- | :--- |
| **模式0** | `NONSEQ` + `SEQ` | 正常传输的地址/数据正确性 |
| **模式1** | `IDLE` | 空闲插入及从机忽略行为 |
| **模式2** | `BUSY` | 突发中 BUSY 插入及地址指向 |
| **模式3** | `BUSY`（非法） | 固定长度突发以 BUSY 结束的违例检测 |
| **模式4** | `NONSEQ` + `SEQ` | 从机插入等待时的传输正确性 |
| **模式5** | `SEQ`（WRAP） | 回环突变的地址边界计算 |
| **模式6** | `IDLE`/`BUSY` + `ERROR` | 错误响应与空操作周期的隔离 |

### 表5：传输类型违例场景

| 违例类型 | 涉及传输 | 违例描述 | 检测优先级 |
| :--- | :--- | :--- | :---: |
| 突发首拍用 SEQ | `SEQ` | 突发第一笔使用 SEQ 而非 NONSEQ | P0 |
| 突发后续用 NONSEQ | `NONSEQ` | 突发第二笔使用 NONSEQ 而非 SEQ | P0 |
| BUSY 作为首拍 | `BUSY` | 突发第一笔就发 BUSY | P1 |
| BUSY 终止固定突发 | `BUSY` | INCR4/WRAP4 最后一拍发 BUSY | P1 |
| IDLE 被当成有效数据 | `IDLE` | 从机错误处理 IDLE 周期 | P0 |
| BUSY 地址指向错误 | `BUSY` | BUSY 周期地址未指向下一拍 | P1 |
| 突发中传输类型跳变 | 混合 | NONSEQ → IDLE → SEQ（中断突发） | P0 |
