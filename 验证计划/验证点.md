# AHB #
## 基础正确性验证 ## 
正常传输：

    其中包括了单次传输和突发传输，突发传输的类型又分为8种，由HBURST控制，其中HBURST为0时就是正常传输
    同时确定HSIZE变量，确保覆盖所有可能的变量，需要确保地址对齐，HSIZE不超过地址总线宽度

有等待传输：

    传输类型一致，但是包含了随机的等待时间，还有边界测试的最长等待时间

## 主机主动插入控制 ##
IDLE插入验证：
        
    在传输完一次传输后不发起新的传输，验证主机是否能正确发起IDLE，且从机是否能正确识别IDLE

突发中BUSY插入验证：
    
    让主机在突发传输中，因暂时无法提供下一笔数据而插入BUSY，验证主机能在不终止突发的前提下，为自己争取额外的准备时间

## 协议边界与错误处理 ##
非法BUSY终止检测：

    主机主动违例，在固定长度突发的最后一拍发送BUSY，验证验证环境本身有能力发现协议违例，同时确保从机有一定程度的容错性

SEQ地址边界验证：

    针对 WRAP 类型突发，专门验证地址在边界处的回绕算法是否正确

IDLE/BUSY 与 ERROR 隔离：

    验证错误响应 (ERROR) 只在有效数据传输时发生，而 IDLE/BUSY 周期不能返回 ERROR
